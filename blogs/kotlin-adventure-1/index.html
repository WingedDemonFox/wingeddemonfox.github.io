Kotlin Adventures 1

This is the beginning of my adventure with Kotlin and Mobile Development.

After years of working as a Microsoft Dynamics CRM/365 developer and teaching myself WPF, I wanted to try something entirely new: Mobile Development.

My first choice was Xamarin, but previous experience in an educational environment wasn’t great and with recent changes in the .NET environment I decided against it.

So I started using Android Studio and Kotlin as programming language. And I immediately regret not taking the mobile course when I had the chance. Oh well.

Java was/is still the main programming language for B.Sc. in Computer Science. Although I worked with Java for a few years now, I am not proficient with it. Which is why I went with Kotlin.

But before I tell you how I did anything without prior knowledge, I want to talk about what my goal was and why I decided to develop an app instead of a desktop application.

I want to have an overview during a battle in DnD with easy access to the following informations:

  • Who’s turn it is
  • Who will be next
  • How many rounds have passed
  • How much time has passed in the gaming world

Since I had no knowledge of how to do anything with Kotlin or Android, I started with the following tutorial: https://developer.android.com/kotlin/get-started. It’s a good place to start to get basic knowledge.

After the tutorial I was already able to create the following proof-of-concept:

  • The app contained seven hardcoded buttons with hardcoded strings.
  • The enabled button indicates the active player.
  • Clicking on the button ends the active players turn and will enable the next one.
  • When the last player had it’s turn, it will increase the number of rounds and time passed and reenable the first one the list.

This was a good start but unusable as everything was hard coded.

But that proof-of-concept showed me three major issues I haven’t thought of before:

  • How do I add new players in general
  • How can I remove players
  • How can I add new players to an ongoing battle

Another thing that I wanted was a list of current players regardless of an ongoing battle. For this to work I used the RecycleView, a concept which allows you to not only to define a list but also how those list-items should look like. There are many different tutorials available regarding the RecycleView. I had to work through multiple to get it working. This was mostly due to the fact I haven’t looked at fragments yet and wanted to only work through one component at the time.

With the RecycleView-Component and more implementations I got the following view:

Next was adding new players to the existing list built for the recycleview. I am not well informed of how to send data from one activity to another. Trying to follow some advice, I gave up and decided to use so called companion object. From what I understand a companion object in Kotlin is something like a global object that every activity can access.

And while setting the app together I stumbled upon another interesting thing: defining the relations of activities inside the AndroidManifest.xml. I honestly expected that I can define those relationships inside the activities and not add it to an extra file.

To be honest: the whole control of defining which button is active and which is not, was at first an absolute hack. But due to having to add buttons dynamically my first hack of simply adding all the buttons to one array and keep track through an index variable. But all in all it didn’t change that much. On the activity the buttons are added into a linearlayout and through that and hoping the order will be kept, I can iterate through the childelements and enable the one I need. Well, first disable all and then enable :D

I was able to use the app during a game and it worked really well and helped me alot.

For me this app is completed. I do have ideas for different changes, but honestly the only thing I think I might do is optimizing the app for smartphone as well and maybe add different color theme to it.

TLDR Those were my first learnings working with Kotlin on mobile development
